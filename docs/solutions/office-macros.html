SignPath for Microsoft Office Macros

Office Macros - Magical Tool for Some, Immense Pain for Others

Magical Tool

Microsoft Office Macros are a very usefull tool to automate workflows in your documents and do boring repetetive tasks very quickly.

You also can interconnect the different Microsoft Office applications together and create unique user experiences for you and your collegues.

Immense Pain

Macros are programs within Office documents that execute with the same permissions as the user opening the document.
That makes Office macros a preferred way for hackers to get into organizations, with attacks ranging from ransomware to spear phishing.
Administrators find it almost impossible to provide policy settings that define which macros may be executed

Use Macro Signing to Control which Macros are allowed to run!

  1. Use group policies to allow execution only for macros signed with trusted certificates
  2. Assign trusted certificates to users and groups
  3. Use SignPath to sign your reviews Office Macros
  4. Get a good night sleep

Use Macro Signing to Con

In SignPath all your organization's code signing certificates are centrally managed. The InfoSec teams have full control over what code is signed and which certificate was used. They can view the entire history in case of an incident and define clear policies to restrict how each certificate may be used. SignPath allows you to import your own certificates or create a certificate signing request (CSR) for your certificate authority (CA). You are also notified on time before your certificate expires.

Other Approaches don't work!

This table shows how readily available policies provide inadequate security and/or impact the business to an unacceptable degree

Method Security level Difficulty Business impact Remarks
Enable macro execution 🔴 🟢 🟢 This should never be enabled
Let users decide whether to execute macros 🟠 🟢 🟢 You cannot rely on users always making the right decision
Disable macro execution except for digitally signed macros 🟡 🟠 🟢 Adequate private key security requires dedicated hardware, plus you need a reliable and auditable process
Disable macro execution except for users who require them 🟡 🔴 🟠 Each of these users still poses a risk, and they often add up
Disable macro execution except for certain storage locations 🟡 🟡 🟡 This will mitigate direct internet/email attacks, but still any user can drop a malicious document in a trusted location
Disable macro execution for everyone 🟢 🟢 🔴 Very safe but often unrealistic
